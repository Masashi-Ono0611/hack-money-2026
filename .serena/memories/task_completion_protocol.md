# Task Completion Protocol

1.  **Verify Implementation:**
    - Run relevant tests (`forge test` for contracts, manual check/lint for frontend).
    - Ensure no regressions.
2.  **Code Quality Check:**
    - Run formatters (`bun run format` / `forge fmt`).
    - Run linters (`bun run lint`).
3.  **Documentation:**
    - Update documentation if features changed.
    - Check specs alignment.
4.  **Commit:**
    - Use Conventional Commits format (e.g., `feat: add new hook`, `fix: resolve overflow`).
    - Descriptive English messages.
